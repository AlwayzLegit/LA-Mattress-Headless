## Detailed Findings

### [HIGH] Sitemap.xml emits all URLs on the apex hostname, forcing a 308 hop per crawl
- File:line: `app/sitemap.ts`
- Problem: sitemap.xml lists 1,096/1,096 URLs on the apex host; apex→www is a 308 so every crawler hit is a redirect.
- Proposed fix: route the sitemap base URL through the canonical `www` host; fix robots.txt `Host:`/`Sitemap:` too.

### [HIGH] `/pages/mattress-store-beverly-hills` is 308 → `/pages/mattress-store-locations` despite being in sitemap at priority 0.85
- File:line: legacy redirect in `data/url-inventory/redirects.json`
- Problem: a legacy Hydrogen-era redirect shadowed the neighborhood route.
- Proposed fix: remove the redirect; the Shopify page is live + published so the neighborhood template renders.

### [HIGH] PLP cross-cut sub-nav (Phase 284) is missing from all sampled collection pages
- File:line: `app/collections/[handle]/page.tsx` + `lib/collection-siblings.ts`
- Problem: the Phase 284 sub-nav was never committed to any branch.
- Proposed fix: build `lib/collection-siblings.ts` + the `<nav class="plp-sibling-nav">` render block + CSS.

### [HIGH] 3 of 5 showroom pages have title and H1 case-normalized identical
- File:line: merchant-set Shopify Admin `global.title_tag` metafields on best-mattress-store-la-brea / best-mattress-store-west-la / mattress-store-in-glendale
- Proposed fix: set distinct SEO titles in Shopify Admin (no code change).

### [MEDIUM] Neighborhood page titles double-brand: " — LA Mattress | LA Mattress"
- File:line: `app/pages/[handle]/page.tsx#generateMetadata`
- Proposed fix: strip a trailing brand suffix before appending the canonical " | LA Mattress".

### [MEDIUM] Neighborhood H1s lack "Los Angeles" / "LA"
- File:line: NeighborhoodPage H1 template
- Proposed fix: build H1 from `neighborhood.name` + ", Los Angeles" (skip when name already contains LA).

### [MEDIUM] Neighborhood H1s render neighborhood names in lowercase
- File:line: NeighborhoodPage H1 string-building
- Proposed fix: use proper-cased `neighborhood.name` directly.

### [MEDIUM] TITLE_MAX=70 truncation cuts brand suffix mid-word on blog/PDP titles
- File:line: `lib/seo.ts#capTitle`
- Proposed fix: drop the brand suffix entirely before truncating the base.

### [MEDIUM] Google/Bing site-verification meta tags absent on every page
- File:line: `app/layout.tsx#generateMetadata` (env-gated)
- Proposed fix: confirm verification method with merchant (DNS/file vs env var). Code is correctly gated; no code defect.

### [MEDIUM] Robots.txt `Host:` and `Sitemap:` directives reference apex, not canonical `www`
- File:line: `app/robots.ts`
- Proposed fix: route through canonical `www` host.

### [MEDIUM] Homepage H1 has typographical error: "Up to60% off."
- File:line: `app/_components/hero.tsx` (adjacent flex-column line spans, no separator in textContent)
- Proposed fix: trailing space on non-last hero lines.

### [LOW] Homepage `<title>` includes "LA Mattress Store — LA Mattress" doubled branding
- File:line: `app/page.tsx` + `app/layout.tsx` (merchant Brand slogan already brand-led)
- Proposed fix: compose-brand-title helper that doesn't double the brand.

### [LOW] `ld-organization` on homepage lacks `sameAs` array
- File:line: `lib/site-config.ts#SOCIAL_PROFILES` (intentionally empty)
- Proposed fix: add merchant-confirmed social profile URLs. Needs merchant input — code already emits `sameAs` when populated.

### [LOW] FAQ on homepage has exactly 6 questions
- File:line: `lib/faq.ts#HOMEPAGE_FAQ`
- Proposed fix: expand to 8–10 questions.
